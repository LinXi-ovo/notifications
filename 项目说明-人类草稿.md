#### 你的卡被拒绝了 cloudfare r2
是银行拒绝的，不是 Cloudflare 不让你用。

原因大概是：

国内银行卡默认关闭境外在线支付 —— 很多银行出于安全考虑，境外网站的扣款默认拦截
银行风控拦截 —— Stripe（美国的支付处理商）发起的 $1 验证扣款，银行系统觉得可疑就给拦了
解决办法：

① 打银行客服电话
跟客服说"我要开通境外在线支付功能"，开通后再试一次就行

② 换备选方案
不折腾卡了，直接用 GitHub 上传——你有 GitHub token，走 API 把文件传到仓库 uploads/ 目录，然后通过 raw.githubusercontent.com 访问，零成本立刻能用

### 🧠 人类知识：HTML button 在表单内的默认行为

**2026-06-05 发现** — 修工具栏闪退时的关键洞察

`<button>` 元素在 `<form>` 内部的默认 `type` 是 **`submit`**，不是单纯的按钮！

这意味着：
- 表单内的任何 `<button>`，点击后会**触发表单提交**
- 表单提交 → 页面刷新/导航 → 编辑器销毁 → 看起来像闪退
- 命令本身能执行（比如加粗成功），但执行后表单提交把页面刷新了

**修复方法**：`<button type="button">` 显式声明为非提交按钮。

**为什么 AI 容易忽略这个**：AI 训练数据中 `type="button"` 的写法虽然常见，但"表单内 button 默认 type=submit"这个细节在 HTML 规范里经常被边缘化。多数 AI 见过的 button 代码片段都是独立使用（不在表单内），所以不会想到要加 `type="button"`。

**以后写代码**：
- 只要 `<button>` 在 `<form>` 内（哪怕嵌套在子组件里），一律 `type="button"`
- 除非你真的想让它提交表单
- 这个规律也适用于 Tiptap 工具栏、模态框内的按钮、搜索框旁的按钮等
