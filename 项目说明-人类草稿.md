### 备用暂存记录

我想到的 5 个要确认的点
1. 🌐 CORS 跨域
前端在 GitHub Pages，数据在 LeanCloud，浏览器会拦截跨域请求。
→ ✅ LeanCloud 国内版支持配置安全域名，需要在控制台把 GitHub Pages 域名加白名单。这个部署时做，不阻碍开发。

2. 🇨🇳 GitHub Pages 国内访问慢
GitHub Pages 在中国有时打不开。
→ ✅ 方案里已经留了 Gitee Pages 双推。建议一开始就创建两个远程仓库，省得后面改。

3. 🧩 Tiptap 自定义节点开发量
音频/视频/PDF 的编辑器内嵌节点需要自己写 ProseMirror 扩展，这部分是整个项目技术难度最大的地方，需要实现节点序列化/反序列化、文件上传流程、播放器渲染。
→ ✅ 已经在 Phase 3 给了 3 天时间，够用。

4. 📱 手机上怎么编辑
Tiptap 在手机上的工具栏和拖放体验有限。
→ ✅ 我的建议：Phase 3 先做桌面端编辑器，移动端编辑可以降级为纯文本+简单工具栏，或者干脆移动端只做浏览和管理，编辑回到电脑上。

1GB 存储：假设每条通知含媒体平均 2MB，能存 ~500 条
2万次 API/天：你+几个同学日常浏览，绰绰有余 → ✅ 如果后面不够了，升级付费版 19 元/月起

#### 你的卡被拒绝了 cloudfare r2
是银行拒绝的，不是 Cloudflare 不让你用。

原因大概是：

国内银行卡默认关闭境外在线支付 —— 很多银行出于安全考虑，境外网站的扣款默认拦截
银行风控拦截 —— Stripe（美国的支付处理商）发起的 $1 验证扣款，银行系统觉得可疑就给拦了
解决办法：

① 打银行客服电话
跟客服说"我要开通境外在线支付功能"，开通后再试一次就行

② 换备选方案
不折腾卡了，直接用 GitHub 上传——你有 GitHub token，走 API 把文件传到仓库 uploads/ 目录，然后通过 raw.githubusercontent.com 访问，零成本立刻能用
