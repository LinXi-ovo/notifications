## 架构概览

```
┌─────────────────────────────────────────────────────────────────────┐
│  通知聚合器 (现有系统)               任务系统 (新增)                  │
│  ┌─────────────────────┐          ┌──────────────────────┐         │
│  │  Bmob 数据层        │          │  localStorage 存储    │         │
│  │  COS 文件存储       │          │  JSON 导入导出        │         │
│  │  Vue Router / Pinia │  ────►   │  Vue Router / Pinia  │         │
│  │  WgEditor / Mermaid │  复用     │  dagre + SVG 渲染    │         │
│  └─────────────────────┘          └──────────────────────┘         │
└─────────────────────────────────────────────────────────────────────┘
```

**核心设计**：任务系统是**独立**于通知系统的模块，但复用其用户系统和编辑器组件。

## 数据模型速查

| 概念 | 文件存储字段 | 说明 |
|---|---|---|
| **Mission** | `mission:{id}` | 项目容器，含 nodes/edges/roles/assignments 等完整数据 |
| **TaskNode** | mission.nodes[] | 任务节点，含状态、完成记录、自定义字段值 |
| **Edge** | mission.edges[] | 依赖边（DAG），含循环检测 |
| **Role** | mission.roles[] | 角色定义，含认领策略（free/approval/password/delegated） |
| **Assignment** | mission.assignments[] | 角色认领记录 |
| **Mission 索引** | `missions:index` | 概要列表 [{ id, title, status, updatedAt }] |

## 关键设计决策

1. **纯前端 SPA + localStorage**：任务数据不经过 Bmob，全部存 localStorage。导入/导出 JSON 文件实现数据迁移。
2. **复用 UserStore**：角色认领的身份来源于通知系统的 `useUserStore`，`username` 作为 userId。
3. **独立路由命名空间**：`/missions/*` 不干扰现有的通知路由 `/` `/detail/:id` 等。
4. **4 种认领策略**：自由认领、审核认领、口令认领、管理员委派，在 Role.claimPolicy 中配置。
5. **3 种完成模式**：single（单人完成）、count（累计 N 人）、all（全员完成）。
6. **dagre + SVG 布局**：使用 dagre 库自动计算 DAG 布局，SVG 绘制边，Vue 组件渲染节点。
7. **循环检测**：添加边时 BFS 检测循环依赖。

- 单独加载预设任务

预设定义在 `src/utils/mission-presets.js`。
