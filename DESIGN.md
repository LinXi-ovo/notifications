# 通知聚合器 — 设计文档

> 大学微信通知聚合器 —— 将分散的微信群通知集中组织和展示

---

## 1. 需求概述

大学微信群通知分散、不集中，关联跳转不便，需要一个自己的网页来良好组织和展示这些通知。

**核心需求**：
- 纯前端静态站点，手动录入通知
- 按类型分类浏览（综测、保研、活动、课程、作业、其他，可自定义）
- 富文本编辑器，图片/音频/视频/PDF 可嵌入在正文任意位置
- 小范围分享给同学（需登录）
- 用户可收藏通知、标记待办（Phase 2）
- 数据可迁移（不锁定平台）

## 2. 架构设计

```
用户浏览器 (Vue 3 SPA)
    │
    ├── UI：Vue 3 + Tailwind CSS
    ├── 编辑器：Tiptap（WYSIWYG 富文本）
    ├── 路由：Vue Router 4
    ├── 状态：Pinia
    │
    └── 数据层：Bmob JS SDK (hydrogen-js-sdk, bmob.cn)
              ├── Notifications 表
              ├── Categories 表
              ├── Files 表（文件注册表）
              ├── Favorites 表
              └── User（内置用户系统）
```

### 为什么选择 Bmob？

| 要求 | 方案 |
|---|---|
| 国内速度快 | ✅ 国内节点 |
| 不需要后端服务器 | ✅ 纯前端 SDK，SPA 直连 |
| 小范围分享权限控制 | ✅ 内置 User + 权限表系统 |
| 免费够用 | ✅ 100MB 存储、2万次/天 API |
| 中文文档 | ✅ 中文官网 doc.bmobapp.com |

## 4. 数据模型

### Category（分类）

管理员可自由增删改，初始分类：综测、保研、活动、课程、作业、其他。

| 字段 | 类型 | 说明 |
|---|---|---|
| `name` | String | 分类名称 |
| `value` | String | 唯一标识符 |
| `icon` | String | Emoji 图标 |
| `color` | String | 主题色 |
| `sortOrder` | Number | 排序权重 |

### Notification（通知）

| 字段 | 类型 | 必填 | 说明 |
|---|---|---|---|
| `title` | String | ✅ | 标题 |
| `content` | String | ✅ | Tiptap 输出的 HTML，媒体内嵌 |
| `type` | String | ❌ | 分类（引用 Category.value） |
| `sourceGroup` | String | ❌ | 来源微信群 |
| `sourcePerson` | String | ❌ | 发布人 |
| `originalLink` | String | ❌ | 原文链接 |
| `priority` | Number | ❌ | 0=普通 1=置顶 2=重要 3=紧急 |
| `tags` | Array | ❌ | 标签列表 |
| `status` | String | ❌ | active / archived |

### File（文件注册表）

上传文件后注册，渲染时通过文件名自动匹配 CDN URL，方便迁移。

| 字段 | 类型 | 说明 |
|---|---|---|
| `name` | String | 文件名（内容中引用用这个名字） |
| `url` | String | Bmob CDN 完整 URL |
| `mimeType` | String | MIME 类型 |
| `size` | Number | 文件大小 |
| `uploader` | Pointer(User) | 上传者 |
| `usedBy` | Array | 被哪些通知引用 |

### Favorite（收藏）

`user` + `notification` 联合唯一，用户收藏通知。

### Todo / TodoProgress（Phase 2）

管理员在通知下创建待办项，用户各自打钩跟踪进度。

## 7. 数据迁移方案

### 核心原则：HTML 内容天然可移植

通知的 `content` 字段是标准 HTML，不绑定任何平台。

### 迁移步骤

```
① 从 Bmob 导出所有表为 JSON（控制台 → 数据表 → 导出）
② 遍历 File 表，下载所有媒体文件到本地
③ 将媒体文件上传到新服务（阿里云OSS/腾讯云COS/自己服务器）
④ 在导出的 JSON 中批量替换 CDN 域名：
   https://xxx.leancloud.cn/ → https://new-cdn.example.com/
⑤ 导入到新系统，完成
```

### 为什么可迁移

- HTML 中的 `<img src="...">` 只是普通 URL，域名可全局替换
- File 注册表记录了所有文件映射，下载+迁移一步到位
- 纯文本替换，不需要解析 HTML
- 导出是标准 JSON + HTML，任何系统都能读
