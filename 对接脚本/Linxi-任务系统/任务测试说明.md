# 任务系统 — 快速测试说明

## 前置条件

1. 启动开发服务器：`npm run dev`（默认 http://localhost:5173）
2. 登录系统（任意账号，管理员更佳）
3. 点击顶部导航栏的 **📋 任务**

---

## 方式一：调试模式一键加载预设（推荐）

### 开启调试模式
1. 点击导航栏 **⚙️ 设置**
2. 找到 **🧪 调试模式** 开关并开启（仅管理员可操作）
3. 回到 **📋 任务**，进入任意任务或新建一个

### 加载预设
在任务图页面顶部会出现 **🧪 调试** 工具栏，包含 5 个预设按钮：

| 按钮 | 测试内容 | 适合验证 |
|---|---|---|
| **📋 班级材料提交** | 5 节点、3 角色、count 完成模式、自定义字段 | 角色认领、多人完成、依赖阻塞 |
| **🏆 评奖评优** | 5 节点、3 角色、含子图标记、JumpLink | 子图标记、权限控制 |
| **✅ 采购审批流** | 5 节点、4 角色、并行审批 count=2 | 并行审批、角色切换 |
| **🧪 单人测试** | 1 节点、single 模式 | 最简流程验证 |
| **🧪 多人测试** | 3 节点、count(3人)+single、依赖边 | count 完成、依赖阻塞/解锁 |

点击按钮 → 自动创建任务并跳转，无需手动搭建。

---

## 方式二：手动创建并测试

### 测试流程

```
1. 进入 📋 任务 → 新建任务（如"测试任务"）
   → 自动带 🟢普通成员 和 🔴管理员 两个角色

2. 点击 "角色" → 添加需要的角色
   [自由认领] 普通成员、执行人
   [审核认领] 班长（需管理员审批）
   [口令认领] 可设置密码
   [管理员委派] 只能由管理员分配

3. 点击 "＋ 添加节点" → 创建任务节点
   - 填写标题
   - 选择负责人角色
   - 选择完成模式:
     • 单人完成: 一人操作即完成
     • 多人完成: 累计 N 人完成
     • 全员完成: 所有认领人完成

4. 点击 "连线" → 在节点间添加依赖边
   → 系统自动检测循环依赖

5. 点击节点 → 底部滑出详情面板
   → "认领此角色" → 认领后可操作
   → 点击状态按钮切换状态
   → 多人模式可点击"标记我已填写"
```

## 各功能模块测试指引

### 1. 角色认领（4 种策略）

| 策略 | 测试步骤 | 预期结果 |
|---|---|---|
| **自由认领** | 点击节点详情 → "认领此角色" | 立即生效，显示"✅ 已认领" |
| **审核认领** | 创建角色时选"审核认领" → 尝试认领 | 提示"已提交认领申请，等待审核" |
| **口令认领** | 创建角色时选"口令认领"并设置密码 → 输入正确口令 | 认领成功 |
| **管理员委派** | 创建角色时选"管理员委派" | 普通用户无法自荐（未实现前端委派面板） |

### 2. 状态流转

| 当前状态 | 可转换到 |
|---|---|
| todo | in-progress |
| in-progress | completed, blocked |
| completed | in-progress（重新打开） |
| blocked | in-progress, cancelled |
| cancelled | todo（恢复） |

### 3. 完成模式

| 模式 | 测试步骤 | 预期结果 |
|---|---|---|
| **single** | 创建节点选"单人完成" → 状态改为 completed | 瞬间完成 |
| **count** | 选"多人完成"目标 N 人 → 多人点击"标记我已填写" | 累计计数，满 N 人自动变为 completed |
| **all** | 选"全员完成" → 所有认领人标记 | 最后一人标记后变为 completed |

### 4. 依赖边

- 添加边时自动检测循环依赖
- 前置节点未完成时后置节点显示 blocked
- 悬停/点击边查看依赖关系

### 5. JSON 导入导出

- **导出**: 任务图页点击「📥 导出」→ 下载 JSON 文件
- **导入**: 任务列表页点击「📥 导入 JSON」→ 选择 JSON 文件

## 预设数据文件

预设定义在 `src/utils/mission-presets.js`，每个预设是一个返回完整 Mission 对象的函数。

添加新预设只需：
1. 在 `mission-presets.js` 中编写新的 `presetXxx()` 函数
2. 在 `ALL_PRESETS` 数组中添加 `{ name, description, fn }` 条目
3. 调试模式自动显示按钮

## 问题排查

| 现象 | 可能原因 | 解决 |
|---|---|---|
| 任务数据丢失 | localStorage 被清 | 重新导入之前导出的 JSON |
| 预设加载后没有自动认领 | 未登录 | 先登录再加载预设 |
| 按钮不可点击 | 不是对应角色 | 先认领该角色 |
| 添加边失败 | 形成循环依赖 | 重新规划依赖关系 |
