# 腾讯云 COS 说明

> COS = Cloud Object Storage（云对象存储）

---

## 什么是对象存储？

传统文件存在电脑硬盘里——有文件夹、树形目录、C 盘 D 盘。

对象存储不一样——它像一个**巨大的公共储物柜**：

```
传统硬盘：     C:\Users\ photos\ 毕业照.jpg
对象存储：    存储桶(bucket) → 对象(object)

没有盘符、没有目录层级、没有「我的文档」
只有「桶」和「对象」
```

### 桶（Bucket）

桶是装文件的容器。可以理解为一个**独立的空间**：

- 一个腾讯云账号可以创建多个桶，比如 `notifications-images`、`notifications-pdfs`
- 每个桶有独立的访问权限、独立的域名
- 桶名全局唯一（整个腾讯云不能重名）

### 对象（Object）

对象就是文件本身。每个对象有：

| 部分 | 说明 | 示例 |
|---|---|---|
| **Key** | 文件名/路径 | `uploads/海报.png` |
| **Data** | 文件内容 | 二进制数据 |
| **Metadata** | 元信息 | 文件类型、大小、上传时间 |

虽然看起来有路径（`uploads/海报.png`），但那只是 Key 的一部分，不是真正的文件夹。

## COS 怎么和我们的项目配合？

```
浏览器（用户）
    │ ① 用户拖入图片/PDF
    ▼
前端（你的 Vue 代码）
    │ ② 调用腾讯云 JS SDK，直传到 COS
    │   不需要自己的服务器，浏览器直传
    ▼
腾讯云 COS（存储桶）
    │ ③ 返回一个公开可访问的 URL
    │   https://xxx.cos.ap-guangzhou.myqcloud.com/海报.png
    ▼
编辑器（Tiptap）
    │ ④ 把 URL 嵌入到通知的 HTML 中
    ▼
其他用户浏览通知
    │ ⑤ 浏览器从 COS 加载图片/PDF
    └─ 和加载普通网页图片一样
```

关键点：**所有操作都在浏览器端完成**，不需要自己的后端服务器。

---

## 为什么 COS 适合我们？

| 需求 | COS 方案 |
|---|---|
| 上传图片 | 浏览器直传，拿到 URL 嵌入内容 |
| 上传 PDF | 同上，支持任何文件类型 |
| 同学浏览通知 | 直接加载 COS URL，和普通网页一样 |
| 迁移数据 | 下载所有文件 → 上传到新服务 → 替换域名 |
| 安全 | 仅管理员可上传，所有人可读取 |

### COS 默认域名可以直接用吗？

可以。COS 默认域名（`xxx.cos.ap-guangzhou.myqcloud.com`）在**浏览器中直接访问完全正常**。之前说的「可能要备案」是指：
- ❌ 在微信/QQ 内部浏览器打开 → 可能被拦截
- ✅ 在普通浏览器（Chrome/Edge）打开 → 完全正常

你的通知网页本身就是普通网页，同学也是在浏览器里打开，**不受影响**。

## 常见疑问

**Q: 对象存储和云硬盘有什么区别？**
云硬盘像电脑的硬盘，必须挂载到云服务器才能用。对象存储可以直接通过 URL 访问，不需要服务器。

**Q: 文件可以删除吗？**
可以。通过 COS 控制台或 API 随时删除不需要的文件。

**Q: 文件被恶意上传怎么办？**
我们只在前端管理后台暴露上传功能（只有管理员能进），普通用户看不到上传入口。

**Q: COS 和 Base64 什么关系？**
现在： 图片转 Base64 → 塞进数据库 → 数据库越来越大
改用 COS：文件直传 COS → 数据库只存 URL → 数据库小、加载快

**Q: 和 Bmob 文件服务比呢？**
Bmob 文件服务需要备案域名才能用，COS 默认域名可以直接用，不需要备案。
