# 任务生成助手 — AI Skill

## 功能

给定一段通知内容（如微信群通知、公告、邮件），AI 分析其内容并生成一个结构化的**任务系统数据**，使接收方能够直接导入任务系统进行跟进。

## 输入

用户提供的通知原文（Markdown 或纯文本），可能包含：
- 任务/待办事项描述
- 截止日期
- 负责人/角色信息
- 前置条件或依赖关系
- 需要多人协作的流程

## 输出格式

输出一个 **JSON 对象**，符合以下 schema。系统会解析此 JSON 并自动创建任务。

### 顶层结构

```json
{
  "title": "任务标题（简明扼要）",
  "description": "任务总体描述",
  "tags": ["标签1", "标签2"],
  "roles": [ ... ],
  "nodes": [ ... ]
}
```

### roles（角色数组）

每个角色定义谁负责做什么。角色名应在 nodes 中引用。

```json
{
  "name": "角色名称（如：普通成员、班长、审核员）",
  "color": "十六进制颜色（如 #3B82F6）",
  "emoji": "Emoji 图标（如 🟢、👨‍🎓）",
  "claimPolicy": "认领策略：free | approval | password | delegated",
  "maxAssignees": 999
}
```

`maxAssignees` 为角色最大认领人数（可选，默认 999）。

### 口令/委派的扩展写法

当 `claimPolicy` 为 `password` 或 `delegated` 时，可使用对象形式：

```json
{
  "name": "班长",
  "color": "#F59E0B",
  "emoji": "👨‍🎓",
  "claimPolicy": { "type": "password", "password": "abc123" },
  "maxAssignees": 3
}
```

| 策略 | 含义 | 使用场景 |
|---|---|---|
| `free` | 自由认领，点击即生效 | 普通成员、执行人 |
| `approval` | 需管理员审批 | 班长、组长等敏感角色 |
| `password` | 输入口令认领 | 线下已约定角色 |
| `delegated` | 管理员委派，不可自荐 | 负责人、审批人 |

### nodes（节点数组）

每个节点是一个具体的工作单元。

```json
{
  "title": "节点标题（动词+宾语，如"填写个人信息表"）",
  "description": "节点描述，说明具体要做什么",
  "content": "详细内容（选填，支持 Markdown 格式）",
  "assignedRole": "负责此节点的角色名称（必须匹配 roles 中的 name）",
  "completionRule": "完成模式：single | count | all",
  "completionTarget": "当 completionRule=count 时，目标完成人数；其他情况填 1",
  "priority": 1,
  "deadline": "截止日期（ISO 格式，选填）",
  "dependsOn": ["前置节点标题列表"],
  "tags": ["标签1", "标签2"]
}
```

`priority` 为节点优先级（可选，0-3，默认 1，数字越大优先级越高）。

**完成模式说明**：

| 模式 | 含义 | 场景 |
|---|---|---|
| `single` | 任意一人完成即完成 | 单人任务 |
| `count` | 累计 N 人完成 | 多人填写信息表 |
| `all` | 所有认领人完成 | 全员确认 |

**依赖关系**：`dependsOn` 中填写前置节点的 `title`，系统会自动转换为依赖边。如果 A 必须在 B 之前完成，B 的 `dependsOn` 应包含 A 的 title。

## 生成原则

1. **忠实于原文**：不要添加原文中没有的角色或步骤
2. **合理拆解**：将原文中的多个步骤拆分为多个节点，每个节点一个明确的任务
3. **明确依赖**：有先后顺序的步骤用 `dependsOn` 表达
4. **合理选择完成模式**：
   - N 人做同一件事 → `count` + `completionTarget=N`
   - 一人完成 → `single`
   - 全员必须做 → `all`
5. **角色精简**：角色数量控制在 1-5 个，避免过度细分
6. **标题简洁**：节点标题控制在 10 字以内，动词开头
7. **合理使用截止日期**：原文中提到的日期才加 `deadline`，不自行编造

## 输出要求

- 只输出 JSON，不含任何前缀/后缀说明文字
- JSON 必须合法（可用 `JSON.parse()` 解析）
- 角色名和 `assignedRole` 必须严格匹配
- `dependsOn` 中的标题必须匹配某个节点的 `title`
