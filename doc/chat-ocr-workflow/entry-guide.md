# 📷 OCR 结果入口 — 使用说明

## 位置

管理后台 → AI 生成（🤖 按钮）→ 切换到 **「📷 OCR 结果」** tab

## 工作流

```
豆包 ← 你发截图
  ↓ 按 skill 格式输出文本
你复制文本
  ↓ 打开 AdminView → AI 生成 → 📷 OCR 结果 tab
你粘贴文本
  ↓ 自动识别标题
你点「📥 填入编辑器」
  ↓ 标题 → 标题栏，正文 → Tiptap 编辑器
你在编辑器里逐个替换 {{图片: xxx}} 占位符
  ↓ 设置分类/优先级/标签
保存
```

## 输入要求

### 格式

**豆包输出的纯文本格式**（不是 HTML）。第一行为标题，后续为正文，段落间用空行分隔。

```
关于期末考试安排调整的通知       ← 第一行 = 标题

各位同学：                       ← 正文开始
                                  ← 空行分隔段落
因教学楼改造工程延期，原定于…
                                  ← 空行
{{图片: 时间安排表.png}}          ← 占位符原样保留
                                  ← 空行
考试地点不变，如有疑问请联系我。
                                  ← 空行
来源：2024级计算机科学班群
发布人：辅导员张老师
```

### 不接受的格式

- HTML 源码格式（`<p>...</p>`）— 会显示为纯文本标签
- 带 ``` 代码块的 Markdown — 会带多余符号进编辑器

> 豆包输出格式由 [skill.md](skill.md) 定义。如果用其他 AI（如 DeepSeek、ChatGPT）生成输入，请确保输出格式一致。

## 字段识别规则

| 字段 | 是否自动识别 | 规则 |
|------|-------------|------|
| **标题** | ✅ 自动 | 取粘贴文本的**第一行非空内容**，自动去掉开头的 emoji（📢📣⭐🔴🚨⚠️✅ 等） |
| **正文** | ✅ 自动 | 从**第二行开始**的全部内容，纯文本转 HTML（空行分隔段落 → `<p>` 标签） |
| **分类** | ❌ 手动选择 | OCR 无法从截图中识别"这是综测还是课程通知" |
| **优先级** | ❌ 手动选择 | OCR 无法判断"重要程度" |
| **标签** | ❌ 手动输入 | 同上 |
| **来源群组** | ❌ 手动输入 | 截图顶部群名可能被裁掉，且不是所有截图都有 |
| **发布人** | ❌ 手动输入 | 同上 |
| **原文链接** | ❌ 手动输入 | 截图里没有可点击的链接 |

### 为什么分类不能自动识别？

你可能会想"那让 AI 在输出里带分类不就行了？"——有两个原因不这么做：

1. **OCR 截图本身不包含"这是综测通知"的标签**，AI 只能靠猜，而猜分类比猜标题容易错得多
2. **分类选择在表单的"更多选项"折叠区域**，用户展开后顺手选一下就行，不增加额外操作

如果以后 DeepSeek 等多模态 API 灰度到可以看图，再考虑把"截图 → 提取文字 → 猜分类"一步完成。

## 占位符说明

`{{图片: 文件名.png}}` 和 `{{文件: 文件名.pdf}}` 等占位符直接保留在正文中，用户粘贴到 Tiptap 编辑器后可以看到它们，然后逐个用实际文件替换。

**替换方法**：
- `{{图片: xxx}}` → 光标定位到占位符位置 → 点工具栏 🖼️ 按钮上传实际图片 → 删除占位符文本
- `{{文件: xxx}}` → 同上，点 📎 按钮上传文件

## 实际效果

| 步骤 | 截图 |
|------|------|
| 粘贴豆包输出 | 输入框自动识别标题行 |
| 点"填入编辑器" | 标题填入标题栏，正文进入编辑器，`{{图片:}}` 原样可见 |
| 替换占位符 | 上传实际图片后删除占位符文本 |
