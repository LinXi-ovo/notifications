# 聊天截图 → 通知内容生成

将微信聊天截图中的通知信息，转换为可直接粘贴到 Tiptap 富文本编辑器中的结构化内容。图片/文件等附件用占位符标记，由用户手动上传替换。

## 使用方法

```
你（用户） → 发送截图给 AI → AI 按本 skill 处理 → 输出结构化文本
                                                      ↓
                                              你复制到 Tiptap 编辑器
                                                      ↓
                                              手动替换 {{占位符}} 为实际文件
```

## 输出格式

### 主格式：Tiptap 可粘贴纯文本

直接生成可粘贴到 Tiptap 可视化编辑器的文本，附件用 `{{占位符}}` 标记：

```text
📢 关于2024-2025学年综合素质测评工作的通知

各位同学好：

2024-2025学年综合素质测评工作现已正式启动，请各位同学按要求提交材料。

⚠️ 重要时间节点：
• 材料提交截止：2026年6月20日
• 班级初审：6月21日-6月25日
• 学院复核：6月26日-6月30日

📋 需要提交的材料：
1. 综测申请表（打印后签名，扫描或拍照提交）
2. 获奖证书复印件
3. 志愿活动证明
4. 如有挂科，需提交补考通过证明

{{图片: 综测申请表填写示例.png}}

{{图片: 加分项对照表.png}}

{{文件: 综合素质测评办法.pdf}}

如有疑问，请联系辅导员张老师（微信：zhanglaoshi）

来源：2024级计算机科学班群
发布人：辅导员张老师
```

### 关于附件占位符

| 占位符 | 含义 | 用户操作 |
|--------|------|----------|
| `{{图片: 文件名.png}}` | 聊天截图中的图片 | 点击 Tiptap 🖼️ 按钮上传实际图片，删除占位符 |
| `{{文件: 文件名.pdf}}` | 截图中的文件/文档 | 点击 📎 上传实际文件，或拖入 |
| `{{音频: 文件名.mp3}}` | 截图中的语音/音频 | 点击 🎵 上传实际音频文件 |
| `{{表格: 描述}}` | 截图中的表格 | 用 Tiptap 表格功能重建，或截图插入为图片 |

占位符放在内容中原本附件出现的位置，用户从上往下处理时自然遇到，不会遗漏。

### 备选格式：Tiptap HTML 源码

如果用户偏好粘贴到 HTML 源码模式（点击 `</>` 按钮切换），可以用 HTML 格式输出：

```html
<p>📢 关于2024-2025学年综合素质测评工作的通知</p>
<p>各位同学好：</p>
<p>2024-2025学年综合素质测评工作现已正式启动。</p>
<p>⚠️ 重要时间节点：</p>
<ul>
  <li>材料提交截止：2026年6月20日</li>
  <li>班级初审：6月21日-6月25日</li>
  <li>学院复核：6月26日-6月30日</li>
</ul>
<!--OCR_PLACEHOLDER: 图片 综测申请表填写示例.png-->
<!--OCR_PLACEHOLDER: 图片 加分项对照表.png-->
<!--OCR_PLACEHOLDER: 文件 综合素质测评办法.pdf-->
<p>来源：2024级计算机科学班群</p>
<p>发布人：辅导员张老师</p>
```

HTML 注释格式的占位符在编辑器中不可见，更适合"先粘贴再替换"的工作流。

## 处理流程

### 第 1 步：接收截图

用户提供 1 张或多张聊天记录截图（支持 png/jpg/webp）。

常见样式：
- 群通知截图（老师发的长图）
- 聊天记录翻拍（手机屏幕拍照）
- 多张截图拼接的连续通知

### 第 2 步：视觉分析

分析截图中包含的内容：

**文字内容提取：**
- 通知标题（通常在开头，大字或加粗）
- 正文段落
- 列表项（带序号 · 的项目）
- 时间、地点、人物等关键信息
- 来源群名称、发布人

**附件识别（用占位符标记）：**
- 截图中的图片/海报 → `{{图片: 描述}}`
- 截图中的文件图标/PDF 预览 → `{{文件: 文件名}}`
- 语音消息图标 → `{{音频: 时长}}`

### 第 3 步：结构化输出

按以下结构组织输出内容：

1. **标题**：通知的标题行（加粗或大字文本）
2. **正文**：段落 + 列表，保持原文逻辑顺序
3. **附件占位符**：插在原文中出现的位置
4. **来源信息**：群组名称、发布人

### 第 4 步：用户确认和替换

用户将生成的文本粘贴到 Tiptap 编辑器后：

```
① 全文粘贴 → 检查内容准确性
② 遇到 {{图片: xxx}} → 点工具栏 🖼️ 上传对应图片
③ 遇到 {{文件: xxx}} → 点工具栏 📎 上传对应文件
④ 调整格式（加粗、列表、标题级别）
⑤ 保存
```

## 处理示例

### 输入：一张微信群通知截图

```
[群名称] 2024级计算机科学班群
```

## 注意事项

1. **保持原意**：不要概括、重写或润色通知内容。提取的文字应忠于原文，只修正 OCR 识别错误（如"锿子"→"孩子"）。

2. **占位符位置准确**：附件占位符应放在它在原文中出现的位置，不要集中放在末尾。用户按顺序处理时自然遇到。

3. **来源信息**：如果截图中有群名称和发布人信息，提取到末尾。如果没有，注明"截图来源"。

4. **多张截图**：如果连续多张截图属于同一条通知，合并为一个输出。如果属于不同通知，分别输出并用 `---` 分隔。

5. **去除冗余**：过滤掉系统消息（"你已添加了xx，现在可以开始聊天了"）、时间戳、群聊中的无关对话。

6. **时间归一化**：如果截图中时间格式不统一（"2026-6-3"、"2026年6月3日"、"昨天"），统一为 ISO 格式。

## App 内快捷入口（方案 B）

在 `AiGenerator.vue` 中增加"📷 OCR 结果粘贴"区域，作为豆包输出的直接接收口。

### 界面布局

```
┌──────────────────────────────────────────┐
│  原始内容 / 📷 OCR 结果  [tab 切换]       │
│                                          │
│  ┌ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ┐   │
│  │ 把豆包生成的文本粘贴到这里         │   │  ← textarea
│  │                                   │   │
│  │ 关于期末考试安排调整的通知         │   │
│  │ 各位同学：                       │   │
│  │ {{图片: 时间表.png}}             │   │
│  │ ...                              │   │
│  └ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ┘   │
│                                          │
│  [自动提取标题] 关于期末考试安排调整的通知  │  ← 取第一行
│  [📥 填入编辑器]                         │  ← 正文→Tiptap，标题→标题栏
└──────────────────────────────────────────┘
```

### 功能逻辑

| 步骤 | 操作 | 说明 |
|------|------|------|
| ① | 粘贴豆包输出 | 用户复制豆包生成的文本，粘贴到 textarea |
| ② | 自动提取标题 | 取首段文字作为标题，显示在预览行 |
| ③ | 点击"填入编辑器" | 正文 → Tiptap 编辑器，标题 → 标题输入框，`{{占位符}}` 原样保留 |
| ④ | 手动替换占位符 | 在编辑器里逐个 `{{图片: xxx}}` → 点 🖼️ 上传实际图片 |
| ⑤ | 设置分类/优先级等 | 手动选择，OCR 无法推断 |

### 相比手动操作的改进

| 步骤 | 无入口（纯手动） | 有入口（B 方案） |
|------|-----------------|-----------------|
| 打开编辑器 | 手动打开通知表单 | AI 自动填入标题 + 正文 |
| 复制的文本量 | 先打开编辑器，再粘贴 | 在 AiGenerator 粘贴后一次填入 |
| 标题提取 | 手动复制标题行 | 自动提取 |
| 附件处理 | `{{占位符}}` 原样保留 | 一样，手动替换 |

改动很小：AiGenerator 加一个 tab 或 textarea，提取第一行当标题，点按钮时把正文 `setContent` 到编辑器、标题赋值到标题栏。准备就绪后可以开始实施。
